## 测试纪律（最优先扫描，对照 SKILL.md 铁律）

- **后补测试**：实现先于测试存在，测试一写就绿——证明不了正确性，只反映实现（铁律 §2）。
- **假测试 / 同义改写**：测试里把实现逻辑同义复述一遍当断言，或断言恒真（`expect(x).toBeDefined()` 占位）；改坏实现也不变红。
- **假红**：测试因导入错/拼写错/缺桩而红，而非断言不满足——RED 失败原因不正确，不算真 RED。
- **照旧实现改编**：重写时留旧代码当参考、照着写测试（铁律 §3 后补测试变体）。
- **skip / 注释 / 弱化断言** 让套件变绿；把期望值改成实际输出。
- **builder 越权改测试/断言/CI** 去变绿（应由 verifier 处理）。
- **mutation 不报警**：把实现某分支改坏，对应测试仍全绿 → 该测试无效，按 bug 处理。

## 通用（语言无关）

- 删/改测试或断言以「变绿」（最严重）。
- 吞异常：`catch {}` 空块、吞掉错误不上报。
- 把 agent 执行流水当文档交付给人。
- 「完成」无证据：未跑构建/测试就声明完成；信任子代理口头自报「过了」而无新鲜输出。
- 边界未覆盖：空值、超大输入、特殊字符、权限不足、并发竞态。
- 错误路径未测：只测 happy path。
- 隐藏假设：依赖未验证的版本/路径/环境，未用工具确认。
- 配置漂移：改了一处配置，未核对相关配置一致性。

## typescript / javascript

- `any` 滥用消解类型检查；`as` 强制断言绕过编译器。
- 漏 `await`，promise 未处理；浮动 promise。
- `==` 与 `!=`（应 `===`）；`==null` 例外需注释。
- 在 React 渲染期产生副作用；useEffect 依赖数组不全。
- 直接改 props/state 而非不可变更新。

## go

- 忽略 `err`（`_ = err` 或不检查返回的 error）。
- goroutine 泄漏：启动未设退出路径；channel 无关闭方。
- 在循环里捕获循环变量（旧版 Go）。
- `defer` 在循环中累积导致资源迟释放。

## python

- 裸 `except:` 吞所有异常。
- 可变默认参数 `def f(x=[])`。
- 未关闭文件/连接（应 `with`）。

## react

- key 用数组下标导致列表 diff 错乱。
- 巨型组件无拆分；渲染期重计算未 memo。
- 受控/非受控混用。
