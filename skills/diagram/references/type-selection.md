# Mermaid 类型选择 — 决策树 / 反例集 / 调色板

主文件给了映射表，这里钉死边界 case、反例与全项目固定调色板。

## 决策树（自顶向下，命中即停）

1. 要表达的是**多方按时间先后发生的交互**（谁先发、谁回谁、有无并发/可选分支）？
   → `sequenceDiagram`（用 `->>` 实线请求、`-->>` 虚线响应、`alt/opt/loop/par` 表条件与并发）。
2. 要表达的是**单个对象/系统在自身状态间的迁移**（idle→running→done、订单状态机）？
   → `stateDiagram-v2`（`[*]` 起止、迁移线带触发事件标签、可嵌套 composite state）。
3. 要表达的是**实体之间的关系与基数**（用户 1—* 订单、表结构与外键）？
   → `erDiagram`（`||--o{` 表基数，列出字段与类型）。
4. 要表达的是**类型/接口的结构与继承组合**？
   → `classDiagram`。
5. 以上都不是，要表达**静态依赖、分层架构、或纯分支决策流程**？
   → `flowchart`（分层用 `subgraph`，判定用菱形 `{}`）。

## 反例集（症状 → 病根 → 正解）

- **症状**：flowchart 里一堆箭头来回指，想表达「客户端发请求、服务端查库再回包」。
  **病根**：把时序当成了静态依赖，生命线和先后顺序全丢了。
  **正解**：`sequenceDiagram`。

- **症状**：flowchart 画订单 `待支付→已支付→已发货→已完成`，还想标「超时则取消」。
  **病根**：这是状态机，flowchart 无法表达「状态」与「触发事件」的语义。
  **正解**：`stateDiagram-v2`，迁移线标事件（`已支付 --> 已发货 : 商家发货`）。

- **症状**：flowchart 画一堆方框代表数据库表，连线表外键。
  **病根**：缺少基数（1:1 / 1:N / N:M）与字段，读者读不出关系强度。
  **正解**：`erDiagram`。

- **症状**：一张 flowchart 既画了微服务依赖，又塞进了一次登录请求的时序。
  **病根**：一图多主题，铁律 §4 被破。
  **正解**：拆两张——架构用 flowchart，登录时序用 sequence。

- **症状**：节点命名 `A/B/C`、连线无标签，「反正我自己看得懂」。
  **病根**：图是给人看的交付物，占位名+裸连线等于没传达信息。
  **正解**：领域词命名 + 每条连线写动作/条件标签。

## 全项目固定四色 classDef（复用，勿每图自创）

```mermaid
%% 四色语义：主路径 / 次要 / 外部依赖 / 告警异常
classDef primary fill:#2563eb,color:#fff,stroke:#1e40af;
classDef minor   fill:#e5e7eb,color:#111,stroke:#9ca3af;
classDef external fill:#f59e0b,color:#111,stroke:#b45309;
classDef alert   fill:#dc2626,color:#fff,stroke:#991b1b;
```

- 主（primary）：关键路径/核心组件。
- 次（minor）：辅助节点。
- 外部（external）：第三方/外部系统边界。
- 告警（alert）：失败分支/异常态/需注意处。

同一项目所有图沿用这套语义与配色，读者跨图一眼对齐含义。

## 渲染自检

落盘前实际渲染一次（Mermaid Live、IDE 插件、或 `npx @mermaid-js/mermaid-cli -i x.mmd -o x.svg`）。报错就修语法，不靠脑补判「应该没错」，更不删报错节点凑通过。
