# Flow：面向 Claude Code 的轻量项目级 Agent 工作流框架 — 技术设计

> 适用运行时：Claude Code
> 面向规模：4–5 人小队的项目级协作
> 设计取向：在保证质量的前提下最大化轻量

Flow 是一个运行在 Claude Code 上的项目级 Agent 工作流框架。它把"流程能力"放进上下文之外的 hooks 常驻在线，让主上下文始终干净；用一个由 Stop hook 驱动的 **Oracle 闭环**，强制"实现"与"独立验证"循环到任务被客观判定通过；用**项目画像**让规范自适应到具体代码库；用**学习闭环**把每次失败沉淀为可复用经验。其核心是一个跨三个时间尺度、互锁的三层反馈控制系统。

本设计按主题拆为七章，建议顺序阅读，各章亦可独立查阅。落地实现速查见 [README](../README.md)。正文沿用统一的 §1–§21 编号，下表给出每章对应的章节区间。

## 术语表

| 术语 | 含义 |
|---|---|
| **tier / R0–R3** | 任务复杂度档。R0 直执 → R3 项目级，决定走多重的流程与多大的循环预算。 |
| **Oracle** | 独立的"完成"裁决者。一条冻结的验证命令，由 hook 在主上下文外运行，agent 不能改它、也不能替它宣布通过。 |
| **L1 / L2 / L3** | 三层反馈回路：任务回路（秒–分）/ 项目回路（天–周）/ 框架回路（周–月）。 |
| **升维** | 同一问题连续失败时，注入更高的认知视角（换眼→升维→归零→移交），而非施压。 |
| **promise** | agent 申请完成的信号 `<promise>FLOW_DONE</promise>`；只是申请，需经 Oracle 裁决。 |
| **黏滞 tier** | 一个 run 内 tier 判一次后保持不变，任务中途消息不重判。 |
| **run** | 一个工作目录对应一个 run，状态存于 `.flow/runs/<cwd 哈希>/`。 |
